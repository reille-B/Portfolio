The context

From paper
to a real system.

Tindela is a fintech operating across four zones in Brazzaville (Centre, Littorale, Nord, Sud). Each zone is run by its own independent distributor, responsible for onboarding the mPOS agents (the merchants who accept the mobile payment terminals) in their zone.

Until now, onboarding meant a paper form. Slow to fill out, hard to read, easy to lose, painful to consolidate, and almost impossible to audit. For a regulated financial product, that's a real liability.

The platform replaces paper with a structured digital file: entered once, controlled, validated, and automatically rendered into the official KYC PDF and the consolidated Excel base. With full traceability of who did what, when.

The solution

A file moves
through three states.

01
Draft
The evaluator (or team lead) captures the file in the field, even fully offline. Resumable later. Private to its author.
→
02
Pending
Submitted for validation. A gate blocks submission while required fields are missing. Duplicate detection (same ID / mPOS # / IMEI) triggers a side-by-side comparison.
→
03
Validated
The validator controls, completes (wave, supervisor, mPOS user #), signs, and the file is locked. Or sent back with a reason. It returns to Draft.
⏻

Offline-first

Everything works without a network: capture, edits, photos, signatures. Local Dexie store, sync queue, optimistic concurrency. A stuck file is flagged, never silently lost.

◈

Team collaboration

A file can be shared with a teammate (the creator stays accountable). Team leads can edit any non-validated file in their team. Conflict guards prevent overwrites.

⊕

Geolocation

One-tap GPS capture, adjustable on a map (street / satellite, free tiles). Each supervisor sees their agents on a territorial map, scoped to their perimeter.

⎙

Official documents

A faithful recreation of the official KYC PDF (identity, engagement, letter, stamps, signatures) via pdf-lib. Consolidated Excel base (numbering, waves) via write-excel-file.

Roles & security

Six roles, strict
separation of duties.

Evaluator
Field agent. Onboards merchants, captures everything.
Team lead
Manages a team of evaluators. Onboards too, helps complete files.
Validator
Controls and validates files in a zone. Produces the final dossier.
Zone manager
Supervises a zone (read + can send back), without validating it.
Administrator
Manages accounts and configuration. No access to KYC files.
Commander
Root of trust. Full rights, designates admins. Break-glass account.
①
RLS at the database level
Postgres Row Level Security is the real barrier: the DB only returns rows the caller is allowed to see. The UI hides what you can't do; the server refuses what you shouldn't.
②
Separation of duties
The person who captures is not the person who validates. The zone manager supervises without validating. Anti-escalation: an admin cannot self-promote to Commander.
③
Per-account isolation, even offline
KYC media are stored separately with their own ACLs. Local IndexedDB stores are isolated per account on the device. Sensitive actions are journaled in an audit log.
Technical choices

Picked to make
offline a first-class citizen.

Frontend
Nuxt 4 (Vue 3.5, SPA) · Nuxt UI 4 · Tailwind CSS v4 · Pinia · VueUse · Zod · TypeScript strict
Backend & data
Supabase (Postgres · Auth · Storage · RLS) · Nitro server routes · service_role for privileged actions
Offline-first core
Dexie (IndexedDB) · custom outbox + push/pull sync · optimistic concurrency · @vite-pwa/nuxt (Workbox)
Maps
Leaflet · OpenStreetMap (street) · Esri World Imagery (satellite). Free, no API key.
Document generation
pdf-lib (server-side official KYC PDF) · write-excel-file (consolidated Excel base, serverless-safe)
Quality & ops
Vitest (74 unit tests on Zod schemas, RBAC, helpers) · pnpm · Vercel (auto-deploy on master)
Why this stack

The data layer is deliberately isolated. The Pinia stores never talk to Supabase directly; they go through a repository pattern. That means we can swap Supabase for a custom REST backend in v2 without touching the pages or stores. Local-first is the foundation; cloud is just one transport.
